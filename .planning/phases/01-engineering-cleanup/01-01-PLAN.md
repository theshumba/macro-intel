  <action>
In `src/services/sourceRegistry.js`, change the `region` field for both the IMF feed (line ~39) and World Bank feed (line ~48) from `'Global Maritime / Strategic Waterways'` to `'Global'`.

Note: `'Global'` is not currently in the REGIONS array in eventModel.js, but this is correct per the CONCERNS.md analysis -- IMF and World Bank are global institutions that should not be classified into a specific geographic region. The `'Global'` value acts as a fallback default region. The geolocation engine will still attempt to resolve a specific region from event text content. If a more precise approach is needed later, this can be revisited. Adding `'Global'` to the REGIONS array in eventModel.js would be a separate decision for a future phase.

Alternatively: Check if eventModel.js REGIONS already has a suitable global entry. If not, set to `'Global'` anyway since `inferRegionFromSource()` in geolocation.js will use this as a default when no country match is found. The region value in sourceRegistry is a default/fallback, not the final classification.

After the fix, run `npm run build` to confirm no issues.
  </action>
