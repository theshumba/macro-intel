<task type="auto">
  <name>Task 2: Implement tiered polling intervals</name>
  <files>
    src/services/ingestionEngine.js
    src/App.jsx
  </files>
  <action>
Currently, `App.jsx` calls `ingestAll()` every 10 minutes for ALL feeds regardless of type. The sourceRegistry already defines `pollMinutes` per feed (15 for official/Tier 1, 10 for news/Tier 2). The requirement is:
- News feeds (Tier 2, pollMinutes: 10): poll every 10 minutes
- Official feeds (Tier 1, pollMinutes: 15): poll every 15 minutes
- Market data (future Tier 4, pollMinutes: 1-5): poll every 1-5 minutes (no market feeds exist yet, but the architecture should support this)

**Implementation approach:**

1. In `src/services/ingestionEngine.js`, add a new exported function `ingestByPollInterval(maxPollMinutes)` that:
   - Uses `getActiveFeedsByPollInterval(maxPollMinutes)` from sourceRegistry.js (already exists -- returns feeds where `pollMinutes <= minutes`)
   - Follows the same pattern as `ingestAll()` but only fetches feeds matching the interval
   - When using the worker path: the worker fetches all feeds at once, so for the worker path, filter the returned items by source name matching the eligible feeds
   - When using the CORS proxy path: only fetch feeds matching the interval
   - Still clusters and deduplicates the returned results
   - Log which tier is being refreshed: `console.info('[ingestion] Tier refresh: feeds with pollMinutes <= ${maxPollMinutes}')`

2. In `src/App.jsx`, replace the single 10-minute `setInterval` with tiered intervals:
   - On mount: call `ingestAll()` once to load everything initially
   - Set up interval 1: call `ingestByPollInterval(10)` every 10 minutes (news feeds)
   - Set up interval 2: call `ingestByPollInterval(15)` every 15 minutes (official feeds -- this also catches news feeds at the 15-min mark, which is fine since they overlap)
   - For future market feeds: a comment placeholder noting that a 1-5 minute interval should be added when market tier feeds are implemented
   - Each interval callback should merge new events with existing state (not replace entirely). Use a merge strategy: new events update existing ones by eventId, new eventIds get appended.
   - Clean up all intervals in the useEffect return function
   - Log each tier refresh with the interval: `console.info('[polling] Refreshing feeds with pollMinutes <= 10')` etc.

   **Important:** The initial `ingestAll()` on mount stays unchanged. The tiered intervals only apply to subsequent refreshes. This avoids the cold-start problem where official data would not load for 15 minutes.

   **Merge strategy for tiered updates:** Since `ingestByPollInterval` returns a subset of events (only from feeds matching the interval), the App.jsx state update should:
   - Take existing events
   - For each new event: if an event with the same `eventId` exists, replace it (updated data); if not, add it
   - Run the combined list through deduplication is NOT needed here since ingestByPollInterval already deduplicates its batch
   - Simply merge by eventId using a Map keyed on eventId

3. Run `npm run build` and `npm run dev` to verify the app starts and the console shows tiered polling messages.
  </action>
